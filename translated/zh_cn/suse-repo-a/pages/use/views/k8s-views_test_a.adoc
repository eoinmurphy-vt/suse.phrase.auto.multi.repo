== 概述

SUSE Observability 对 Kubernetes 及其组件有着深入的了解。在集群中安装 SUSE Observability 代理后，它会自动检测 Kubernetes 应用程序的拓扑结构并将其可视化。这包括构成应用程序的 Kubernetes 资源，如部署、pod、服务和入口。它还会自动检测和可视化构成集群的 Kubernetes 基础设施拓扑，如节点、命名空间和持久卷。

SUSE Observability 为以下 Kubernetes 原生资源提供了专门的概述和亮点页面：

image::k8s/k8s-menu.png[Kubernetes 视图]

所有其他 Kubernetes 资源都能在拓扑视图中被识别和可视化。

== 概览页

概览页面提供环境中特定 Kubernetes 资源的高级概览。这些概述突出显示了有关资源的最重要信息，包括其健康状况、年龄、命名空间、群集等。

image::k8s/k8s-service-overview.png[概览页]

在概览页面，您可以点击 Kubernetes 资源之一，导航到该资源的亮点页面。

== 重点页面

亮点页面显示特定 Kubernetes 资源最重要的信息概览。在这里，您可以看到资源的健康状况、资源上激活的监控器以及资源上发生的事件。此外，我们还会显示资源的关键指标。

image::k8s/k8s-pod-highlights.png[重点页面]
