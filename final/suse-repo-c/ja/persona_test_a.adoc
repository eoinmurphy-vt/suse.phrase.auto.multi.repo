= 主要なペルソナとその役割
:revdate: 2025-05-30
:page-revdate: {revdate}

[cols="1,2,2", options="header"]
|===
|ペルソナ |責任 |主なメリット

|プラットフォーム・エンジニア
a|* クラスタ間での{product_name} インストールのセットアップと管理。
* マルチクラスタのアプリケーションとポリシーの展開を定義し、実施する。
* GitRepo と`fleet.yaml` の設定を管理する。
* クラスタ全体のアップデートとロールバックを実行する。
a|* クラスタ間の集中制御。
* 標準化とコンプライアンスの向上。

|アプリケーション開発者
a|* アプリケーションHelmチャートまたはマニフェストをGitで構造化する。
* `fleet.yaml` でデプロイメントターゲットと環境を定義する。
* UI または CLI を使用してデプロイのステータスを監視します。
a|* GitOpsベースのワークフロー。
* より速く、自動化され、環境間で一貫性のあるデプロイメント。

|セキュリティ管理者
a|* セキュリティポリシーをコードとして定義し、実施する。
* セキュリティツールや設定を導入する。
* クラスタ全体のセキュリティ態勢を監査し、管理する。
a|* リスクの低減とコンプライアンス管理の一元化
* 自動化された一貫性のあるセキュリティ設定。
|===
