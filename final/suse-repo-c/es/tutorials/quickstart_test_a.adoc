== Instale

{product_name} se distribuye como un gráfico Helm. Helm 3 es una CLI, no tiene ningún componente del lado del servidor, y su uso es bastante sencillo. Para instalar Helm 3 CLI siga las https://helm.sh/docs/intro/install[instrucciones de instalación oficiales de].

[CAUTION]
.{product_name} en Ranchero
====
Rancher tiene cuadros de mando separados para {product_name} y utiliza un repositorio diferente.
====


[tabs]
====
Linux/Mac::
+
[source,bash]
----
brew install helm
helm repo add fleet https://rancher.github.io/fleet-helm-charts/
----

Windows::
+
[source,bash]
----
choco install kubernetes-helm
helm repo add fleet https://rancher.github.io/fleet-helm-charts/
----
====

Instale las cartas {product_name} Helm (hay dos porque separamos los CRD para una mayor flexibilidad).

[source,bash]
----
helm -n cattle-fleet-system install --create-namespace --wait fleet-crd \
    fleet/fleet-crd
helm -n cattle-fleet-system install --create-namespace --wait fleet \
    fleet/fleet
----

== Añadir un repositorio Git a Watch

Cambie `spec.repo` al repositorio git de su elección.  Los archivos de manifiesto de Kubernetes que deben desplegarse deben estar en `/manifests` en su repositorio.

[source,bash]
----
cat > example.yaml << "EOF"
apiVersion: fleet.cattle.io/v1alpha1
kind: GitRepo
metadata:
  name: sample
  # This namespace is special and auto-wired to deploy to the local cluster
  namespace: fleet-local
spec:
  # Everything from this repo will be run in this cluster. You trust me right?
  repo: "https://github.com/rancher/fleet-examples"
  paths:
  - simple
EOF

kubectl apply -f example.yaml
----

== Obtener estado

Conocer el estado de la flota

[source,shell]
----
kubectl -n fleet-local get fleet
----

Deberías ver algo como esto crearse en tu cluster.

----
kubectl get deploy frontend
----
