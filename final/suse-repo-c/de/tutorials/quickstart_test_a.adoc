== Installieren Sie

{product_name} wird als Helm-Chart verteilt. Helm 3 ist eine Befehlszeilenschnittstelle (CLI), die keine serverseitige Komponente hat, und seine Verwendung ist recht einfach. Zur Installation von Helm 3 CLI folgen Sie den https://helm.sh/docs/intro/install[offiziellen Installationsanweisungen von].

[CAUTION]
.{product_name} in Rancher
====
Rancher hat separate Steuerkarten für {product_name} und verwendet ein anderes Repository.
====


[tabs]
====
Linux/Mac::
+
[source,bash]
----
brew install helm
helm repo add fleet https://rancher.github.io/fleet-helm-charts/
----

Windows::
+
[source,bash]
----
choco install kubernetes-helm
helm repo add fleet https://rancher.github.io/fleet-helm-charts/
----
====

Installieren Sie die {product_name} Helm-Karten (es gibt zwei, da wir die CRDs für ultimative Flexibilität getrennt haben).

[source,bash]
----
helm -n cattle-fleet-system install --create-namespace --wait fleet-crd \
    fleet/fleet-crd
helm -n cattle-fleet-system install --create-namespace --wait fleet \
    fleet/fleet
----

== Git Repo zur Überwachung hinzufügen

Ändern Sie `spec.repo` in das Git-Repository Ihrer Wahl.  Kubernetes-Manifestdateien, die bereitgestellt werden sollen, sollten unter `/manifests` in Ihrem Repository zu finden sein.

[source,bash]
----
cat > example.yaml << "EOF"
apiVersion: fleet.cattle.io/v1alpha1
kind: GitRepo
metadata:
  name: sample
  # This namespace is special and auto-wired to deploy to the local cluster
  namespace: fleet-local
spec:
  # Everything from this repo will be run in this cluster. You trust me right?
  repo: "https://github.com/rancher/fleet-examples"
  paths:
  - simple
EOF

kubectl apply -f example.yaml
----

== Status abrufen

Status der Flottenaktivitäten abrufen

[source,shell]
----
kubectl -n fleet-local get fleet
----

Sie sollten sehen, dass in Ihrem Cluster etwas Ähnliches erstellt wird.

----
kubectl get deploy frontend
----

----
NAME       READY   UP-TO-DATE   AVAILABLE   AGE
frontend   3/3     3            3           116m
----
