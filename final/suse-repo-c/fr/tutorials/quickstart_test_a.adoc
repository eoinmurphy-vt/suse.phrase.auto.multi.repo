== Install

{product_name} est distribué sous la forme d'une carte Helm. Helm 3 est un CLI, il n'a pas de composant côté serveur et son utilisation est assez simple. Pour installer le CLI Helm 3, suivez les https://helm.sh/docs/intro/install[instructions d'installation officielles de].

[CAUTION]
.{product_name} dans Rancher
====
Rancher a des diagrammes de barre distincts pour {product_name} et utilise un dépôt différent.
====


[tabs]
====
Linux/Mac::
+
[source,bash]
----
brew install helm
helm repo add fleet https://rancher.github.io/fleet-helm-charts/
----

Fenêtres::
+
[source,bash]
----
choco install kubernetes-helm
helm repo add fleet https://rancher.github.io/fleet-helm-charts/
----
====

Installez les cartes {product_name} Helm (il y en a deux car nous séparons les CRD pour une plus grande flexibilité).

[source,bash]
----
helm -n cattle-fleet-system install --create-namespace --wait fleet-crd \
    fleet/fleet-crd
helm -n cattle-fleet-system install --create-namespace --wait fleet \
    fleet/fleet
----

== Ajouter un Git Repo à Watch

Changez `spec.repo` pour le repo git de votre choix.  Les fichiers manifestes de Kubernetes qui doivent être déployés doivent se trouver à l'adresse `/manifests` dans votre répertoire.

[source,bash]
----
cat > example.yaml << "EOF"
apiVersion: fleet.cattle.io/v1alpha1
kind: GitRepo
metadata:
  name: sample
  # This namespace is special and auto-wired to deploy to the local cluster
  namespace: fleet-local
spec:
  # Everything from this repo will be run in this cluster. You trust me right?
  repo: "https://github.com/rancher/fleet-examples"
  paths:
  - simple
EOF

kubectl apply -f example.yaml
----

== Obtenir le statut

Obtenir l'état d'avancement des activités de la flotte

[source,shell]
----
kubectl -n fleet-local get fleet
----

Vous devriez voir quelque chose comme ceci créé dans votre cluster.

----
kubectl get deploy frontend
----

----
NAME       READY   UP-TO-DATE   AVAILABLE   AGE
frontend   3/3     3            3           116m
